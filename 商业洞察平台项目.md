# 商业洞察平台 (InsightPro) — 完整产品文档

> 版本：v1.0 | 更新日期：2026-06-04

---

## 一、产品定位

**InsightPro** 是一个面向**云服务商业市场（中长尾、腰部行业客户）业务领导**的 AI 驱动商业洞察平台。

核心价值：帮助业务领导快速获取行业动态、竞争分析、政策法规、市场机会，支撑日常商业决策。

---

## 二、技术架构

### 2.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **前端框架** | Next.js 14+ (App Router) + React 18+ + TypeScript | SSR/SSG 混合渲染 |
| **样式系统** | Tailwind CSS v4 + CSS 变量主题 | 设计令牌驱动 |
| **字体** | Mona Sans（正文）+ Cormorant Garamond（标题） | Claude 同款正文字体 |
| **图标** | Lucide React | 统一图标库 |
| **可视化** | Recharts | 折线/柱状/面积/饼图 |
| **后端框架** | FastAPI (Python 3.10+) | 异步 API 服务 |
| **AI 引擎** | DeepSeek-V3 (OpenAI 兼容 API) | 结构化报告生成 |
| **数据库** | SQLite (trending.db) + Supabase (PostgreSQL) | 本地缓存 + 云端持久化 |
| **ORM** | Prisma ORM | 数据模型管理 |
| **定时任务** | APScheduler (Asia/Shanghai 时区) | 每日 09:00 自动刷新 |
| **数据爬虫** | BeautifulSoup + Requests | 百度热搜 / GitHub Trending |

### 2.3 API 端点清单

| 方法 | 路径 | 功能 | 数据源 |
|------|------|------|--------|
| GET | `/` | 健康检查 | — |
| GET | `/api/baidu-hotsearch` | 百度实时热搜（降级 SQLite） | 百度 top.baidu.com |

| GET | `/api/github-trending/history?days=7` | 历史 Trending 查询 | SQLite |
| POST | `/api/github-trending/refresh` | 手动触发刷新 | — |

| GET | `/api/dashboard/stats` | 数据大屏指标 | 硬编码 |
| GET | `/api/daily-insight` | 今日洞察 | 硬编码 |
| POST | `/api/tasks/analyze` | 提交 AI 分析任务 | DeepSeek API |

## 三、页面功能清单

### 3.1 首页 `/`（今日商业洞察）

**定位**：平台门面，每日商业简报入口

**功能模块**：
1. **Hero 区域**：深色背景 + 背景图片 + 当日日期 + "进入数据大屏"按钮
2. **看板导航**（7 格）：行业覆盖、热点追踪、友商监控、政策法规、商业机会、实时快讯、AI 分析 — 点击跳转子版块
3. **市场洞察看板**（2×3 网格）：6 个子模块预览卡片，每个展示 3 条摘要，点击进入详情页
4. **右侧栏**：
   - 百度实时热搜（自动刷新，5 分钟轮询）
   - 今日数据概览（任务数/报告数/AI 调用）
   - 系统状态（AI/数据库/爬虫）

### 3.2 行业全景 `/insights/industry`

**定位**：6 大行业商业动态监测

**覆盖行业**：生物医疗、交通、基础设施、互联网、零售、制造

**功能**：
- 6 张行业卡片，含图标、状态标签、摘要、外部信源链接
- 底部 CTA 跳转案例库

### 3.3 案例库 `/insights/industry/cases`（三级页面）

**定位**：面向中长尾/腰部客户的上云最佳实践

**功能**：
- 6 个行业案例（制造/零售/医疗/金融/教育/政务）
- 每个案例包含：客户背景、痛点、解决方案、效果数据、交付周期
- 行业筛选按钮
- 底部"预约架构师咨询"CTA

### 3.4 技术热点 `/insights/hotspots`

**定位**：GitHub Trending 实时监控 + 历史追溯

**功能**：
- 今日/本周/月三个维度切换
- 项目卡片：仓库名、描述、语言、Star 数、Fork 数、今日增星
- 历史记录面板：近 14 天每日快照，点击切换查看
- 手动刷新按钮
- 数据来源标识（实时/本地存储）

### 3.5 友商洞察 `/insights/competitors`

**定位**：场景化竞争分析（华为云 vs 5 大友商）

**覆盖友商**：AWS、Microsoft Azure、阿里云、腾讯云、火山云

**6 大场景**：
1. 中资企业出海（vs AWS + Azure）
2. AI 大模型开放平台（vs Azure + AWS）
3. 开发者生态争夺（vs AWS + Azure + 阿里云）
4. 企业 SaaS 生态（vs Azure + 阿里云）
5. 腰部客户性价比（vs 阿里云 + 火山云）
6. 传统行业数字化（vs 阿里云 + 腾讯云）

**每个场景包含**：
- 对手标识 + 市场规模
- 3 项具体差距（精确到友商能力数据 + 严重等级 + 业务影响）
- 4 条可执行落地策略
- 紧迫度时间窗口

### 3.6 政策法规 `/insights/policy`

**定位**：影响中长尾/腰部客户云采购决策的核心政策追踪

**覆盖政策**：
1. 等保 2.0（强制执行，高影响）
2. 信创替代（政策加速，高影响）
3. 数据出境安全评估（需持续关注，中影响）
4. OPC 海外云服务合规（新规酝酿，中影响）
5. 行业数字化补贴（多省推行，高影响）
6. SOC 安全运营资质（行业趋势，中影响）

**每个政策包含**：概要 → 对业务的影响 → 建议行动 → 关键时间节点

**底部商机洞察**：近期商机窗口 + 华为云优势 + 需关注风险

### 3.7 商业快讯 `/insights/news`

**定位**：全球核心商业媒体头条

**数据源**：Reuters、Bloomberg、财新网、Wired

**功能**：4 张新闻卡片，含来源、日期、标题、摘要、阅读全文链接

### 3.8 商业机会 `/insights/opportunities`

**定位**：不同层级客户的市场切入与增长建议

**目标客户**：
1. 腰部客户（中大型企业）— 订阅制安全合规服务
2. 长尾客户（个体/小微）— 个体工商户自动化套件
3. 初创企业 — 分布式 AI 算力众包

**每个机会包含**：目标客户、标题、引言、实战建议、外部参考链接

### 3.9 数据大屏 `/dashboard`

**定位**：全局数据监控与可视化

**功能**：
- 4 个 KPI 卡片（总分析任务/活跃用户/报告生成/数据覆盖率）
- 4 个 Recharts 图表：
  - 折线图：技术趋势指数（云服务/AI算力/边缘计算/大数据）
  - 柱状图：本周活跃度
  - 面积图：累计增长趋势
  - 饼图：行业分布
- 时间范围切换（本周/本月/本季）
- 实时监测看板：预警监控 + 服务健康度 + 实时流量

### 3.10 深度研报 `/reports`

**定位**：AI 生成的结构化商业研报展示

**功能**：
- 报告头部：标题、日期、分享/导出 PDF 按钮
- 左栏：市场情绪指数（3 个进度条）+ 核心结论提炼（3 张卡片）
- 右栏：社论式长文（执行摘要 → 市场格局 → 战略建议）

### 3.11 历史日报 `/history`

**定位**：按日期查阅过往商业洞察日报

**功能**：
- 搜索框（按日期/标签/关键词）
- 时间线卡片：日期标记、摘要标签、数据统计、当日重点高亮
- 点击"查看当日完整报告"跳转首页

### 3.12 系统设置 `/settings`

**定位**：平台配置管理

**功能**：
- 用户信息（用户名/邮箱/套餐）
- 通知设置（4 项开关：每日日报/友商预警/政策更新/系统告警）
- 数据源配置（GitHub/百度/DeepSeek/Supabase 连接状态）
- 安全设置（API Key/数据库连接）

## 四、设计规范

### 4.1 设计风格

**定位**：现代商务风格，沉稳专业

**参考**：Bloomberg Terminal、McKinsey 报告、Claude 官网

**设计原则**：
- 白底 + 浅灰边框 + 深色文字 — 干净克制
- 极少量色彩点缀 — 仅用于状态标识和分类
- 大量留白 — 信息密度适中，不拥挤
- 细边框（1px）+ 柔和阴影 — 不抢眼
- 商务深色 hero — 稳重感

## 五、数据流

```
用户访问页面
    ↓
前端 (Next.js) → fetch API → 后端 (FastAPI:8000)
    ↓                              ↓
静态页面 / SSR              爬虫实时抓取 → 成功 → 返回数据 + 存入 SQLite
                                    ↓ 失败
                              从 SQLite 读取历史数据 → 返回降级数据
```

**百度热搜**：实时抓取 → 存入 `baidu_hotsearch` 表 → 失败降级读库

**GitHub Trending**：实时抓取 → 存入 `github_trending` 表 → 每天 09:00 定时任务自动刷新 → 支持历史查询

## 八、开发规范

1. **安全第一**：API Key 零硬编码，统一通过 `.env` 管理，`.env` 不入库
2. **模块化**：数据抓取、AI 提示词、前端 UI 彻底分层
3. **错误处理**：所有 API 请求 try-catch，提供友好错误提示
4. **响应式**：Tailwind CSS 响应式布局，移动端可用
5. **版本管理**：SemVer，重大更新记录在 `log/versions.md`
